What is Fabric.css?

A brief introduction to the framework

Fabric.css is a pure CSS3 framework built using the principles of Google Material Design.


Fabric.css was built with usability and performance in mind. Being in early stages of development, it still lacks some core functionality. Also, it isolates its classes as much as possible to make it compatible with other frameworks and CSS libraries, but if those libraries use !important to override CSS values used by Fabric.css, that might be a problem.


Want to play with the code on CodePen? Here's a link!


Why use Fabric.css?

Unlike many other CSS frameworks, Fabric.css is lightweight, highly modifiable, consistent and both desktop and mobile friendly. It is also open source (more in "Licensing").


Dependencies


The main stub includes links to all the dependencies used by the project. At this point, those are:

  • Normalize.css
  • Material Icons Font

Dependencies


Fabric.css is highly modular. Each module performs a certain function and might be replaced or removed if the user wishes.

  • fabric.css
    The main stub. It imports the dependencies, the modules and applies some global overrides.
  • fabric--layout.css
    Layout adjustments. The 12 column grid, component widths, heights, spacing, ett.
  • fabric--elevation.css
    Elevation-related data. Shadows and z-index.
  • fabric--palette.css
    Colors used by Fabric.css.
  • fabric--typography.css
    Typography-related data.
  • fabric--animation.css
    Animation speeds for component groups.
  • fabric--components.css
    Data related to the individual component styles, where consistency is not required.

Documentation


Sadly, there is none at the moment. There's a demo index.html file included with Fabric.css you can take a look at. Once Fabric.css is fully functional, I'll build a complete documentation around it.


The code itself is fairly simple and easy to understand and modify.
